# Discord Bot

This service is the Discord-facing product surface for SteamWeb.

It is a slash-command bot that delegates business logic to the backend API and focuses on:

- command handling
- Steam account connect flow initiation
- recommendation response rendering
- daily digest posting

## Commands

### `/login`

Starts the Steam account connection flow.

Behavior:

- requests a Steam connect link from the API
- sends the link back to the user
- polls for completion
- posts the linked Steam profile in the channel when the flow succeeds

### `/nenchoigi`

Requests recommendations based on:

- genre
- mood
- session length
- budget
- relevance mode

The bot returns:

- ranked recommendations
- recommendation reasons
- Steam store links
- YouTube gameplay links
- compact Steam and Reddit review context

### `/digestnow`

Admin-only command that posts the current daily Steam digest in the active server channel.

## Daily Digest

The bot supports scheduled daily posting for these sections:

- Most Played
- Trending
- Hot Releases
- Popular Releases
- New This Week
- Releases Today

The payload is generated by the backend API.

## Responsibilities

The bot is intentionally thin.

It should own:

- Discord interactions and embeds
- slash command registration and sync
- daily posting loops
- channel messaging behavior

It should not own:

- recommendation ranking
- Steam account business logic
- data aggregation
- long-term state management

Those responsibilities stay in the API and data layers.

## Local Setup

### Prerequisites

- Python 3.11+
- running backend API
- configured `.env`

### Install

```bash
pip install -r requirements.txt
```

### Run

```bash
python bot.py
```

## Required Environment Variables

- `DISCORD_BOT_TOKEN`
- `DISCORD_CLIENT_ID`
- `BOT_SERVICE_TOKEN`

One of:

- `BACKEND_BASE_URL`
- `PUBLIC_API_BASE_URL`

Recommended:

- `DISCORD_GUILD_ID`
- `STEAM_REDIRECT_URI`
- `DAILY_DIGEST_ENABLED`
- `DAILY_DIGEST_CHANNEL_ID`
- `DAILY_DIGEST_HOUR_UTC`
- `DAILY_DIGEST_MINUTE_UTC`

## Development Notes

- the bot calls backend endpoints directly
- most responses are ephemeral where appropriate
- guild sync is useful during development to avoid slow global command propagation
- the current implementation is gateway-based and assumes an always-on runtime

## Deployment Notes

### Current Hosted Model

- designed to run continuously on Cloud Run
- relies on an always-on process for gateway connectivity and scheduler loops

See:

- [docs/gcp-api-bot-deploy.md](D:\secret\steamweb\docs\gcp-api-bot-deploy.md)

### Low-Cost Direction

The long-term low-cost direction is to move away from a full always-on gateway dependency where practical and prefer:

- Discord interaction endpoints
- HTTP-driven command execution
- smaller scheduled tasks

This does not invalidate the current bot, but it is the architectural direction for budget-constrained deployment.
